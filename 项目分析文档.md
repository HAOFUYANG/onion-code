# Onion Code 项目分析文档

> 本文档基于对源代码的逐文件阅读与分析整理而成，覆盖项目架构、核心模块、工具系统、技能系统、安全设计与配置管理。

## 一、项目定位

Onion Code 是一个基于 **LangChain + LangGraph** 的 CLI AI Agent 工具，核心理念是「工具调用 + 技能扩展」的工程化能力。它以电影《星际穿越》中的 TARS 机器人为人格原型（幽默 75%、诚实 90%），在终端内提供流式对话、代码执行、文件操作、网络检索与可插拔技能系统。

- **入口命令**：`onionCode`（`bin/onionCode.js` → `dist/agent/cli.js`）
- **技术栈**：TypeScript + LangChain 生态 + better-sqlite3 + commander/inquirer/chalk
- **默认模型**：DeepSeek（`deepseek-v4-flash`），兼容 OpenAI 接口

## 二、整体架构

```
用户终端
  │
  ▼
cli.ts ──────────────► input.ts（TTY 输入 / Slash 命令面板）
  │                    slash_commands.ts（/config /sessions /rewind ...）
  │
  ▼
agent.ts（LangGraph Agent + 流式输出 + AbortSignal 中断）
  │
  ├──► tools.ts（9 个工具：search/read_file/write_file/exec/run_js/run_py/web_search/web_fetch/load_skill）
  │       └── security.ts（三层安全防护，被 exec/run_js/run_py/write_file 共享）
  │
  ├──► skills.ts（技能发现 / 加载 / 注入 System Prompt）
  │       └── skills/*/SKILL.md（planner / pdf / skill-creator / efficiency-report / travel-guide）
  │
  ├──► SqliteSaver（.data/checkpointer.db，按 thread_id 持久化会话）
  │
  └──► config.ts + python_env.ts（配置中心 + Python 虚拟环境管理）
```

## 三、核心模块详解

### 3.1 CLI 入口与交互（`src/agent/cli.ts`）

基于 `commander` 构建两种运行模式：

- **`ask <message>`**：单轮问答，直接流式输出后退出（`cli.ts:54-70`）
- **默认交互模式**：进入 `startInteractiveChat()` 循环（`cli.ts:79-185`），维护 `threadId`（UUID），支持 ESC 中断

**ESC 中断机制**（`cli.ts:116-145`）：进入 raw 模式监听 stdin，检测到 `0x1b`（ESC）时调用 `abortController.abort()`，流式输出循环中通过 `signal?.aborted` 提前 break。

**错误格式化**（`cli.ts:16-51`）：针对 5 类常见错误提供友好提示——DeepSeek 内容安全审查、API Key 无效（401）、额度不足（429）、LangGraph 递归限制、网络超时。

### 3.2 输入处理（`src/agent/input.ts`）

`readUserInput()` 区分 TTY 与非 TTY 两种模式：

- **TTY 模式**（`input.ts:138-260`）：启用 `readline.emitKeypressEvents` + raw 模式，自实现一个轻量级 Slash 命令面板。支持：
  - `↑/↓` 选择候选命令
  - `Tab` 补全选中命令
  - `Esc` 关闭面板
  - `Ctrl+U` 清空输入
  - `Backspace` 删除字符
  - `Enter` 提交（Slash 命令解析参数，普通文本作为消息）
- **回退模式**（`input.ts:22-47`）：非 TTY 环境用 `readline.question` 问答式输入

面板渲染（`renderSlashPanel`）使用 `chalk` 高亮选中项（橙色背景），底部显示快捷键提示。

### 3.3 Slash 命令系统（`src/agent/slash_commands.ts`）

定义统一的 `SlashCommand` 接口（name / aliases / description / handler），内置 7 个命令：

| 命令 | 别名 | 功能 |
|------|------|------|
| `/config` | `/conf` | 打开配置中心（inquirer 交互） |
| `/rewind <id>` | `/rw` | 切换到指定历史会话 |
| `/sessions` | `/sess` | 查看最近 20 条会话（表格渲染） |
| `/new` | - | 新建会话 |
| `/theme` | - | 占位（未实现，TARS 式提示） |
| `/help` | - | 打印所有命令 |
| `/exit` | `/quit` | 退出程序 |

匹配逻辑（`matchSlashCommands`）：前缀匹配 name 或 aliases，大小写不敏感。

### 3.4 Agent 核心（`src/agent/agent.ts`）

**模型配置**（`agent.ts:67-74`）：
```typescript
new ChatOpenAI({
  model: process.env.OPENAI_MODEL ?? "deepseek-v4-flash",
  apiKey: process.env.OPENAI_API_KEY,
  configuration: { baseURL: "https://api.deepseek.com/v1" },
  streaming: true,
})
```

**System Prompt 构建**（`agent.ts:20-54`）：`buildSystemPrompt()` 拼接两部分：
1. TARS 人格设定（幽默/诚实参数、说话风格、工具使用规则——明确禁止 `write_file + exec` 跑代码）
2. `getSkillText()` 返回的技能清单

**Agent 创建**（`agent.ts:77-92`）：`createAgent` 注入 9 个工具 + checkpointer。

**流式执行**（`agent.ts:102-141`）：`runAgentStream` 使用 `streamMode: "messages"`，逐 chunk 遍历，跳过非 AI 消息和 tool_call_chunks，只回调 `content` 文本。`recursionLimit: 100` 控制最大步数。

**会话持久化**（`agent.ts:59-64`）：`SqliteSaver.fromConnString` 在 `.data/checkpointer.db` 创建 SQLite 检查点数据库，通过 `configurable.thread_id` 续接历史。

### 3.5 会话管理（`src/agent/sessions.ts`）

直接用 `better-sqlite3` 只读访问检查点数据库：

- **`querySessions(limit)`**（`sessions.ts:60-135`）：SQL 联查 `writes` 表，取每个 thread 最早一条 user 消息，按最近活跃度（max checkpoint_id）倒序。从 `checkpoint_id`（UUIDv7）提取时间戳转为相对时间。
- **`threadExists(id)`**（`sessions.ts:44-57`）：校验 thread_id 是否存在，供 `/rewind` 使用。
- **`renderSessionsTable`**（`sessions.ts:138-171`）：用 `cli-table3` 渲染 thread_id / 最后输入 / 时间三列表格。

## 四、工具系统

### 4.1 工具清单（`src/agent/tools.ts`）

统一导出 9 个工具，均基于 `@langchain/core/tools` 的 `tool()` 函数 + `zod` schema 定义：

| 工具 | 文件 | 功能 |
|------|------|------|
| `search` | `search.ts` | 示例工具，返回天气（仅演示） |
| `read_file` | `read_file.ts` | 读取当前目录内文件 |
| `write_file` | `write_file.ts` | 写入当前目录内文件 |
| `exec` | `exec.ts` | 执行 Shell 命令 |
| `run_js` | `run_js.ts` | 执行 JavaScript/Node.js |
| `run_py` | `run_py.ts` | 执行 Python |
| `web_search` | `web_search.ts` | Tavily 网络搜索 |
| `web_fetch` | `web_fetch.ts` | 抓取网页内容 |
| `load_skill` | `load_skill.ts` | 加载技能完整内容 |

### 4.2 安全设计（`src/agent/tools/security.ts`）

三层安全防护，被 `exec` / `run_js` / `run_py` / `write_file` 共享：

**第 1 层：危险命令黑名单**（`exec.ts:8-65`）
覆盖 rm/del/mv/cp/dd/mkfs/format/shutdown/sudo/chmod/kill/wget/curl/tar 等 40+ 高危命令。

**第 2 层：Eval 注入检测**（`exec.ts:69-77`）
阻止 `node -e`、`python -c`、`ruby -e`、`perl -e`、`php -r`、`deno eval`、`bun -e` 等通过解释器注入任意代码。

**第 3 层：危险 API 模式检测**（`security.ts:4-22`）
正则扫描代码内容，阻断：
- Node.js：`fs.rm/rmSync/unlink/rmdir`、`fs.writeFile`、`fs.chmod/chown`、`child_process.*`、`exec/spawn`、`require('fs')`/`require('child_process')`
- Python：`shutil.rmtree/move/copy`、`os.remove/unlink/rmdir/system`、`subprocess.run/call/Popen`、`pathlib.Path.unlink/rmdir`

**路径安全**（`read_file.ts:12-16`、`write_file.ts:13-17`）：用 `path.relative` 校验，禁止路径逃逸到 cwd 之外（`..` 开头或绝对路径）。

### 4.3 代码执行工具

**`run_js`**（`run_js.ts`）：
1. 检查 Node.js 可用性
2. 安全扫描阻断危险 API
3. 写入系统临时目录（`os.tmpdir()`）的 `.js` 文件，文件名含时间戳+随机串
4. `execSync` 执行，15s 超时，512KB 缓冲
5. `finally` 块清理临时文件

**`run_py`**（`run_py.ts`）：
1. 调用 `getPythonForCode()` 选择运行时（虚拟环境优先，按代码 import 动态安装缺失包）
2. 安全扫描
3. 写临时 `.py` 文件，`spawnSync` 执行（`shell: false`），15s 超时
4. 清理临时文件

### 4.4 网络工具

**`web_search`**（`web_search.ts`）：封装 `@langchain/tavily` 的 `TavilySearch`，需 `TAVILY_API_KEY`，返回最多 3 条结果。

**`web_fetch`**（`web_fetch.ts`）：
- 只允许 http/https 协议（`isValidUrl` 校验）
- `fetch` + `AbortController` 实现 15s 超时
- 响应体上限 512KB
- 针对性错误处理：AbortError / ENOTFOUND / ECONNREFUSED / ECONNRESET

### 4.5 文件工具

- **`read_file`**：路径校验 → 判断是否目录 → UTF-8 读取
- **`write_file`**：路径校验 → 目录判断 → **内容危险 API 扫描** → UTF-8 写入

## 五、技能系统

### 5.1 技能发现与加载（`src/agent/skills.ts`）

**目录解析**（`getSkillsDir`，`skills.ts:32-47`）：三级 fallback——`__dirname/skills` → `__dirname/../src/agent/skills` → primary。

**Frontmatter 解析**（`parseFrontmatter`，`skills.ts:14-28`）：正则提取 `---` 之间的 YAML，匹配 `name:` 和 `description:` 字段。

**三个核心函数**：
- `discoverSkills()`：遍历 skills 目录子目录，返回 `SkillInfo[]`（name + description + dir）
- `loadSkill(name)`：按 name 定位，返回完整 SKILL.md 内容
- `getSkillText()`：拼接技能清单文本，注入 System Prompt，引导 AI 用 `load_skill` 工具按需加载

### 5.2 技能清单

项目内置 5 个技能：

| 技能 | 定位 | 关键特性 |
|------|------|----------|
| `planner` | 待办/计划/日程 | 多种输出格式（todo list / 项目计划 / 日程表），优先级分级 |
| `pdf` | PDF 处理 | pypdf/pdfplumber/reportlab，合并/拆分/旋转/水印/加密/OCR/表单 |
| `skill-creator` | 技能创建与评测 | 完整 eval 流程：测试用例 → 基准对比 → 评估查看器 → 描述优化 |
| `efficiency-report` | 效能度量报告 | Excel/CSV → DORA 对标 → Chart.js 可视化 HTML 报告 |
| `travel-guide` | 旅行路书 | 行程规划/美食/住宿/交通，结构化 Markdown 输出 |

### 5.3 技能加载机制（`src/agent/tools/load_skill.ts`）

`load_skill` 工具先调用 `discoverSkills()` 验证技能存在（提供友好错误），再 `loadSkill()` 返回完整内容。这是「渐进式披露」模式——System Prompt 只注入 name + description（约 100 词），完整内容按需加载。

## 六、配置与 Python 环境

**配置结构**：

AppConfig {
  python: {

- 持久化到 `.data/config.json`
- `mergeConfig()` 做浅层合并，保证默认值兜底
- `/config` 命令通过 `inquirer` 交互式修改镜像源、自动安装开关，可选立即初始化环境

### 6.2 Python 环境管理（`src/agent/python_env.ts`）

**Python 探测**（`getPythonCandidates`，`python_env.ts:42-56`）：跨平台候选列表——macOS/Linux 查 `python3` / `/usr/bin/python3` / `/opt/homebrew/bin/python3` / `/usr/local/bin/python3`；Windows 查 `python` / `py -3`。

**虚拟环境**（`ensureVenv`，`python_env.ts:76-107`）：检查 venv 是否存在 → 找基础 Python → `python -m venv` 创建（120s 超时）→ 缓存路径。

**依赖安装**（`python_env.ts:127-159`）：
- `getMissingPackages()`：用 `importlib.util.find_spec` 检测缺失包
- `detectRequiredPackages()`（`python_env.ts:172-187`）：正则扫描代码 import 语句，pandas→自动加 openpyxl
- `installPackages()`：`pip install` + 镜像源参数（180s 超时）
- `autoInstall` 为 false 时直接报错，提示用 `/config` 开启

**运行时选择**（`getPythonForCode`，`python_env.ts:189-222`）：有 import → 检测安装 → 否则用 venv → 否则用缓存 → 否则用系统 Python。

## 八、关键设计决策与观察

### 8.1 工具使用约束

System Prompt 明确规定（`agent.ts:47-50`）：
- **禁止** `write_file + exec` 跑代码，必须用 `run_py` / `run_js`（自动处理临时文件和清理）
- `write_file` 仅用于创建用户要保留的文件
- `exec` 仅用于 shell 命令，不用于跑脚本

这是为了防止临时文件残留和命令行转义问题。

### 8.2 会话持久化的巧妙利用

`sessions.ts` 直接读取 LangGraph 的 `writes` 表（而非通过 SDK 查询），用 SQL 联查提取会话列表。从 UUIDv7 的前 48 位提取 Unix 毫秒时间戳，无需额外存储时间字段——这是一个很轻量的设计。

### 8.3 安全策略的权衡

安全检测基于正则模式匹配，优点是零依赖、快速；局限是可能误报（如代码中字符串包含 `subprocess`）或漏报（变体写法）。对于 CLI Agent 场景，这种「够用且透明」的策略是合理的折中。

### 8.4 技能系统的渐进式披露

三级加载避免上下文膨胀：
1. **元数据**（name + description）始终在 System Prompt 中
2. **SKILL.md body** 通过 `load_skill` 按需加载
3. **打包资源**（scripts/references/assets）按需读取

这与 Claude 官方 skill-creator 的设计哲学一致。

### 8.5 模型耦合

`agent.ts:71` 硬编码了 DeepSeek 的 baseURL。若要切换模型提供商，需修改源码。可改进为通过环境变量 `OPENAI_BASE_URL` 配置。

## 九、构建与运行

```bash
# 安装依赖
pnpm install

# 开发模式（ts-node 直接运行）
pnpm dev

# 构建（编译 TS + 复制 skills 目录）
pnpm build

# 生产运行
pnpm start  # 或直接 onionCode

# 测试
pnpm test   # vitest
```

构建脚本（`package.json:16`）会 `cp -r src/agent/skills/* dist/agent/skills/`，保证技能资源在产物中可用。`skills.ts` 的多级 fallback 也兼容了 skills 未复制到 dist 的情况。

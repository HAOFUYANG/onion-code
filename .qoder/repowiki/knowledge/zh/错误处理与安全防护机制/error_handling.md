该项目采用**防御性编程**与**安全沙箱**相结合的错误处理策略，核心目标是保障 CLI Agent 在执行系统命令和代码时的安全性与稳定性。

### 1. 核心策略：安全优先的容错处理
- **安全拦截即“错误”**：在工具执行前，通过多层黑名单（危险命令、Eval 模式、危险 API）进行静态扫描。一旦触发，直接返回 `Error:` 开头的字符串，而非抛出异常。这种设计将安全风险转化为可控的业务反馈。
- **静默失败与降级**：对于非关键路径（如配置加载、技能发现、临时文件清理），采用 `try-catch` 捕获后忽略或回退到默认值，确保主流程不因次要故障中断。
- **结构化错误输出**：所有工具函数统一返回包含 `Error:` 前缀的字符串，便于上层 UI 识别并展示给用户。

### 2. 关键实现细节
- **工具层 (Tools)**：
  - `exec.ts` 和 `run_py.ts` 均实现了超时控制（30s/15s）和缓冲区限制，防止资源耗尽。
  - 捕获 `spawnSync` 或 `execSync` 的异常，优先提取 `stderr`，其次 `stdout`，最后才是系统错误消息。
- **环境管理 (Python Env)**：
  - `python_env.ts` 定义了 `PythonEnvironmentResult` 接口，通过 `ok` 字段标识状态。在 venv 创建、包安装等环节均返回明确的 `error` 描述。
- **会话持久化 (Sessions)**：
  - `sessions.ts` 在操作 SQLite 数据库时严格使用 `try-finally` 确保连接关闭，防止文件锁死。

### 3. 开发者规范
- **禁止裸抛异常**：Agent 调用的工具函数应始终返回结果字符串，内部消化所有运行时异常。
- **统一错误前缀**：向用户展示的错误信息必须以 `Error:` 开头，以便前端组件（如 `ToolLog.vue`）进行样式区分。
- **资源清理兜底**：涉及临时文件（如 `run_py` 生成的 `.py` 文件）的操作，必须在 `finally` 块中执行清理逻辑。
- **安全左移**：在执行任何外部命令或代码前，必须先调用 `hasDangerousApi` 或相关安全检查函数。

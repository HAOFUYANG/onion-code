### 1. 核心策略：工具级隔离与用户友好提示
该代码库采用**“工具内部消化 + CLI 层统一格式化”**的错误处理模式。核心逻辑（Agent 引擎）不直接抛出原始异常，而是由各个 Tool（如 `exec`, `run_py`）在内部通过 `try-catch` 捕获执行错误，并将其转换为结构化的 `Error: ...` 字符串返回给 LLM。这种设计确保了 Agent 流程的稳定性，使其能够根据错误信息自动尝试修复或向用户反馈。

### 2. 关键实现细节
- **CLI 层错误映射 (`src/agent/cli.ts`)**：
  - 定义了 `formatError` 函数，将底层技术异常（如 `401`, `ETIMEDOUT`, `Recursion limit`）映射为简洁、易懂的中文提示。
  - 针对 DeepSeek API 特有的安全拦截（`Content Exists Risk`）进行了专门识别。
- **工具级健壮性 (`src/agent/tools/*.ts`)**：
  - **执行类工具** (`exec.ts`, `run_py.ts`, `run_js.ts`)：均包含超时控制（15s-30s）、缓冲区限制以及详细的退出码/信号检查。执行失败时，优先返回 `stderr` 内容。
  - **网络类工具** (`web_fetch.ts`)：实现了 DNS 查找失败、连接拒绝、重置及超时等多种网络异常的细分处理。
  - **资源清理**：使用临时文件执行代码的工具（`run_py`, `run_js`）均在 `finally` 块中执行清理逻辑，防止临时文件泄漏。
- **安全防御即错误处理**：
  - **多层过滤**：在 `exec.ts` 和 `security.ts` 中实现了黑名单命令、Eval 模式注入及危险 API 调用的正则匹配。一旦触发，立即返回阻断错误信息，从源头防止系统级破坏。

### 3. 运行时适配与中断处理
- **流式中断 (`src/ink/runtime/adapter.ts`)**：在 Ink 界面运行时，利用 `AbortSignal` 处理用户的 ESC 中断请求。适配器内部维护了 token 队列和状态锁，确保在中断发生时能优雅地停止流式输出并释放资源。
- **数据库只读保护 (`src/agent/sessions.ts`)**：在查询历史会话时，以 `readonly: true` 模式打开 SQLite 数据库，并在 `finally` 块中确保连接关闭，防止因并发写入导致的锁竞争错误。

### 4. 开发者规范
- **禁止透传原始堆栈**：所有面向用户或 Agent 的错误信息必须经过脱敏和格式化，严禁直接暴露 `err.stack`。
- **工具返回值标准化**：自定义 Tool 在执行出错时，应返回以 `Error:` 开头的字符串，以便 Agent 能够准确识别任务失败状态。
- **资源生命周期管理**：涉及文件系统或网络连接的操作，必须遵循“申请-使用-清理”的闭环模式，优先使用 `try-finally` 结构。
